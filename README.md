P2P ファイル一時保管庫 (P2P Drop)

ブラウザ間で直接ファイルを送受信する、サーバーレスの一時ファイル共有アプリケーションです。WebRTC技術（PeerJS）を利用しており、ファイルが外部のクラウドサーバー等に保存されることはありません。

✨ 特徴

完全なローカル・P2P通信: ファイルは送信者のデバイスから受信者のデバイスへ直接転送されます。高いプライバシーを保てます。

自動消去（有効期限）機能: 設定した時間（5分〜3時間）が経過すると自動的に通信を遮断し、ブラウザのメモリからファイル参照を破棄して共有を停止します。

動画・大容量ファイル対応: データを細かく分割（チャンク化）して送信するため、動画などのファイルも転送可能です。受信後はブラウザ上でプレビュー再生ができます。

シンプル構成: バックエンドの構築は不要で、1つのHTMLファイルで構成されています（※P2P通信の確立にはPeerJSの公開シグナリングサーバーを使用しています）。

🚀 使い方

送信者（ファイルを渡す側）

本アプリ（index.html）をブラウザで開きます。

共有したいファイルをドラッグ＆ドロップ、またはクリックして選択します。

自動消去までの時間（有効期限）を選択します。

「共有リンクを作成」ボタンをクリックします。

生成されたリンクをコピーし、受信者にメールやチャット等で送ります。

【重要】受信者がファイルのダウンロードを完了するまで、絶対にこのタブ（画面）を閉じないでください。 タブを閉じると通信が切断され、共有が即座に停止します。

受信者（ファイルを受け取る側）

送信者から送られてきた共有リンクをブラウザで開きます。

自動的に送信者と接続され、ファイルの受信が開始されます。

プログレスバーが100%になり、「受信完了！」と表示されるまでそのまま待ちます。

プレビューを確認し、「デバイスに保存」ボタンをクリックしてファイルを保存します。

⚠️ 動作環境とホスティングについて

このアプリはURLパラメータ（?peer=...）を利用して通信相手を特定します。そのため、HTMLファイルを直接ダブルクリックして開いた状態（URLが file:///... の状態）では、別の端末からアクセスできる共有リンクを生成できません。

別のスマホやPCと共有したい場合は、以下のいずれかの方法でアプリをご利用ください。

Webホスティングサービスを利用する（推奨）
GitHub Pages、Vercel、Netlify、Cloudflare Pages などの無料静的ホスティングサービスに index.html をアップロードして公開します。これが最も確実で簡単な方法です。

ローカルサーバーを利用する（同一ネットワーク内のみ）
送信者と受信者が同じWi-Fi（LAN）に接続している場合は、ローカルサーバーを立ち上げてアクセスします。

例: VS Codeの拡張機能「Live Server」を使用する

例: ターミナルで python -m http.server 8000 を実行し、http://[あなたのローカルIPアドレス]:8000/ でアクセスする

🛑 制限事項

メモリ使用量: 受信側はブラウザのメモリ（RAM）上でファイルの断片を結合してファイルを復元します。そのため、数GBに及ぶ極端に巨大なファイルを受信すると、特にスマートフォンのブラウザなどがクラッシュする可能性があります。一時的な動画やドキュメントの共有用途を想定しています。
